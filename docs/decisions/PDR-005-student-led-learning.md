# PDR-005 - Student-Led Learning

## Date

July 10, 2026

## Status

Accepted

## Volume / Chapter Affected

Product Vision, Student Platform, Mentor Platform, Organization Platform, AI Coach, Portfolio, Certificates.

## Decision

YUVA Club is built around student-led learning. Students learn by presenting, practicing, asking questions, teaching peers, reflecting, volunteering, and growing through active participation.

## Reason

Leadership and communication skills are built through practice, not passive content consumption. The platform should encourage students to take action and build confidence through repeated learning moments.

## Alternatives Considered

- Build YUVA Club primarily as a content library.
- Make adults or mentors the center of every learning workflow.
- Focus only on certificates rather than the learning journey.

## Consequences

- Student workflows must prioritize action and reflection.
- Dashboards should surface next meaningful student actions.
- Presentations, practice mode, peer learning, and feedback are core features.
- AI should recommend next practice steps, not only summarize performance.
- Portfolios should capture the learning journey, not just final artifacts.

## Developer Impact

- Student dashboard should answer: what should I do next?
- UX should minimize friction between login and next learning action.
- Do not bury presentations, practice, or feedback deep in navigation.

## Follow-Up Work

- Define Student Dashboard chapter.
- Define Presentation Center chapter.
- Define Practice Mode chapter.
- Define AI Coach chapter.
